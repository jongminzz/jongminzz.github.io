# 알고리즘 풀이 포스팅 스킬

## 목적
알고리즘 문제의 링크와 풀이 코드를 받아서, 문제 분석 + 핵심 포인트 + 풀이 코드를 담은 블로그 포스트를 생성한다.

## 블로그 경로
- 프로젝트 루트: `/Users/ijongmin/workspace/blog`
- 글 저장 위치: `content/post/{slug}/index.md`
- 스타일 가이드: `/Users/ijongmin/workspace/blog/.claude/style-guide.md`

## 글쓰기 스타일
- **어투**: `-했다`, `~이다` 체 (style-guide.md 준수)
- **톤**: 간결하고 핵심적. 불필요한 설명 없이 요점만.
- **AI 표현 금지**: style-guide.md의 AI 표현 회피 규칙 준수

## 입력
사용자에게 다음 두 가지를 받는다:

1. **문제 링크** (필수): LeetCode, 백준, 프로그래머스 등
2. **풀이 코드** (필수): Python 코드

둘 다 한 번에 주면 바로 진행. 하나라도 빠지면 자연스럽게 요청한다.

## 실행 흐름

### 1. 문제 정보 수집

**링크가 주어진 경우**:
- WebFetch 또는 브라우저 도구로 문제 페이지에 접속하여 다음을 추출:
  - 문제 제목
  - 문제 설명
  - 제약 조건 (입력 범위)
  - 예제 입출력
  - 난이도 (있으면)
  - 알고리즘 분류/태그 (있으면)

**링크 접속이 안 되는 경우**:
- 사용자에게 문제 내용을 직접 붙여넣기 요청

### 2. 풀이 코드 분석

사용자가 제공한 코드를 분석하여:
- 사용된 알고리즘/자료구조 파악
- 시간 복잡도 계산
- 공간 복잡도 계산

### 3. 포스트 작성

아래 템플릿 구조로 글을 작성한다:

```markdown
---
title: "[{플랫폼}] {문제 제목}"
date: {YYYY-MM-DD}
draft: false
categories:
    - Algorithm
tags:
    - {알고리즘 분류}  # 예: binary-search, dp, graph, greedy 등
    - {플랫폼}         # 예: leetcode, boj, programmers
    - {난이도}         # 예: easy, medium, hard, silver-3 등
---

## 문제

{문제 요약 2-3줄}

- **제약 조건**: {핵심 제약 조건}
- **링크**: [{플랫폼} - {문제 제목}]({URL})

## 핵심 포인트

- {제약 조건 분석: N의 범위가 X이므로 O(?) 이하로 풀어야 한다}
- {이 문제의 핵심은 XX이다}
- {접근 방법: 왜 이 알고리즘을 선택했는지}
- {다른 접근이 가능하면: 다른 방법과의 비교}

## 풀이

\```python
{사용자 제공 코드}
\```

## 복잡도

- **시간**: O(?)
- **공간**: O(?)
```

### 4. 글 작성 세부 규칙

**문제 요약**:
- 문제 전체를 옮기지 않는다. 핵심만 2-3줄로.
- 예제 입출력은 포함하지 않는다 (링크에서 볼 수 있으므로).

**핵심 포인트**:
- 제약 조건에서 시간복잡도를 추론하는 과정을 보여준다.
  - 예: "N이 최대 10^5이므로 O(N log N) 이하로 풀어야 한다"
  - 예: "N이 최대 20이므로 비트마스킹이나 완전 탐색이 가능하다"
- 이 문제의 핵심 아이디어를 명확하게 짚는다.
- 가능하면 "왜 이 접근이 되는지"를 짧게 설명한다.
- 다른 접근 방법이 있으면 간단히 언급한다 (깊게 파지 않는다).

**풀이 코드**:
- 사용자가 제공한 코드를 그대로 사용한다.
- 코드를 임의로 수정하지 않는다.
- 필요하면 코드 위에 한두 줄 설명을 추가할 수 있다.

**복잡도**:
- 시간/공간 복잡도를 간단히 기재한다.
- 필요하면 왜 그런 복잡도인지 한 줄 설명을 추가한다.

### 5. 사용자 확인

작성한 포스트를 보여주고 피드백을 받는다:
- 핵심 포인트가 맞는지
- 추가하고 싶은 설명이 있는지
- 수정할 부분이 있는지

### 6. 파일 저장

**slug 생성**:
- `{플랫폼}-{문제번호 또는 제목}` 형식
- 예: `boj-1920`, `leetcode-two-sum`, `programmers-42862`

**저장 위치**: `content/post/{slug}/index.md`

**완료 메시지**:
```
알고리즘 풀이가 저장되었습니다!

파일: content/post/{slug}/index.md
문제: [{플랫폼}] {문제 제목}
알고리즘: {분류}
난이도: {난이도}

배포할까요? (/blog-publish)
```

## 특별 케이스

### 여러 풀이를 비교하고 싶은 경우
```
사용자: "이 문제 두 가지 풀이로 풀었는데 비교해줘"
```
→ 풀이 섹션을 "풀이 1", "풀이 2"로 나누고 각각의 복잡도를 비교

### 문제 링크 없이 코드만 주는 경우
```
사용자: "백준 1920번이야" + 코드
```
→ 문제 번호로 URL 구성하여 접속 시도

### 코드에 주석이 없는 경우
- 코드에 주석을 임의로 추가하지 않는다.
- 핵심 포인트 섹션에서 로직을 설명한다.

## 실행 시 주의사항

1. **style-guide.md 필수 로드**: 매 실행마다 읽을 것
2. **코드를 수정하지 않는다**: 사용자가 제공한 코드를 그대로 사용
3. **과도한 설명 금지**: 간결하게. 핵심만.
4. **제약 조건 → 시간복잡도 추론은 반드시 포함**: 이것이 이 스킬의 핵심 가치
5. **문제 전체를 복사하지 않는다**: 요약만. 원문은 링크로.
